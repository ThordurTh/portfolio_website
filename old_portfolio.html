Portfolio Website - Exam Project

A portfolio website made in the end of my first semester of multimedia design studies.

Website

Description

The website was intended as an exam project to demonstrate all the skills learned that semester while displaying the solution to each assignment. That includes creating a design, making a prototype, writing semantic HTML, style with CSS and be able to communicate the process.

Process

The website was intended to be informative and a showcase of previous assignments so I settled on a minimal design. It uses a limited set of monochromatic colours and simple serif fonts. For the most part I went with a flat design in regards to sub-page navigation and icons. The main navigation however has a sort of a neu-morphism design that, looking back, I would reconsider to keep the overall design more cohesive.

In terms of coding the website, it is purely HTML, CSS and a few lines of JS for navigation functionality. I made sure to use relevant semantic HTML and to make my life easier I split each page up into separate CSS files. Each sub-page makes use of CSS grid to position elements and includes a navigation container on the left that follows when scrolling. My Javascript knowledge was quite limited at the time so the only functionality is toggling the “Portfolio” button which displays a dropdown list.

Conclusion

The website is the result of 4 months of learning the subject and serves as a comparison between recent projects where I make more use of Javascript, have a better understanding of responsiveness and optimisation and how to make use of databases and frameworks.
